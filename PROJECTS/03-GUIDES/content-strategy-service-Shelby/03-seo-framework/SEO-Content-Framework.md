## 4. Content Optimization Guidelines

### 4.1 Title Optimization

**Formulas That Work:**
```
1. How to [Achieve Result] Without [Common Obstacle]
2. [Number] [Adjective] Ways to [Desired Outcome]
3. The Ultimate Guide to [Topic]
4. [Topic]: Everything You Need to Know
5. Why [Common Belief] Is Wrong (And What Works Instead)
6. [Number] [Topic] Mistakes That Are Costing You [Result]
7. [Year] Guide to [Topic]: [Specific Benefit]
8. How [Company/Person] [Achieved Result] in [Timeframe]
```

### 4.2 Introduction Formula (APP Method)

```
AGREE: State something the reader agrees with
PROMISE: Tell them what they'll learn
PREVIEW: Give a brief outline of what's coming

Example:
"Creating content that ranks is hard. (AGREE)
In this guide, you'll learn the exact framework we use 
to rank content on page 1. (PROMISE)
We'll cover keyword research, on-page optimization, 
and promotion strategies. (PREVIEW)"
```

### 4.3 Content Depth Guidelines

| Content Type | Minimum Word Count | Ideal Word Count | Key Elements |
|--------------|-------------------|------------------|--------------|
| Blog Post | 1,000 | 1,500-2,500 | Headers, images, CTA |
| Ultimate Guide | 3,000 | 5,000-10,000 | TOC, multiple sections, resources |
| Landing Page | 500 | 800-1,500 | Benefits, social proof, CTA |
| Product Page | 300 | 500-1,000 | Features, specs, reviews |
| Case Study | 1,000 | 1,500-3,000 | Problem, solution, results |
| List Post | 1,500 | 2,000-4,000 | Clear items, examples |
| How-To Guide | 1,500 | 2,000-5,000 | Steps, screenshots, tips |

## 5. Technical SEO Requirements

### 5.1 Page Speed Targets

| Metric | Target | Tool to Measure |
|--------|--------|-----------------|
| Largest Contentful Paint (LCP) | < 2.5 seconds | PageSpeed Insights |
| First Input Delay (FID) | < 100 milliseconds | PageSpeed Insights |
| Cumulative Layout Shift (CLS) | < 0.1 | PageSpeed Insights |
| Time to First Byte (TTFB) | < 600 milliseconds | WebPageTest |
| Fully Loaded Time | < 3 seconds | GTmetrix |

### 5.2 Image Optimization

| Format | Use Case | Compression |
|--------|----------|-------------|
| WebP | General use (modern browsers) | 80-85% quality |
| JPEG | Photos, complex images | 75-85% quality |
| PNG | Graphics with transparency | Optimize colors |
| SVG | Icons, logos, simple graphics | Minify |

**Image Size Guidelines:**
- Full-width hero: 1920px wide, < 200KB
- Blog post images: 1200px wide, < 150KB
- Thumbnails: 400px wide, < 50KB
- Icons: SVG or < 50px PNG

### 5.3 Mobile Optimization Checklist

- [ ] Responsive design (test on multiple devices)
- [ ] Touch-friendly buttons (minimum 44x44px)
- [ ] Readable font size (minimum 16px body)
- [ ] No horizontal scrolling
- [ ] Fast mobile load time (< 3 seconds)
- [ ] Mobile-friendly navigation
- [ ] Tap-to-call phone numbers
- [ ] Accelerated Mobile Pages (AMP) if applicable

## 6. Content Refresh Strategy

### 6.1 When to Update Content

| Signal | Action | Priority |
|--------|--------|----------|
| Traffic declining >20% MoM | Refresh content | High |
| Ranking dropped 5+ positions | Update + re-optimize | High |
| Information outdated (>1 year) | Update facts/stats | Medium |
| New competitors ranking higher | Analyze + improve | Medium |
| New keyword opportunities | Add sections | Medium |
| Seasonal content | Annual refresh | Scheduled |

### 6.2 Content Refresh Process

```
1. AUDIT
   - Review analytics (traffic, rankings, engagement)
   - Identify what's working/not working
   - Check for outdated information

2. RESEARCH
   - Current top-ranking content
   - New keywords to target
   - Fresh data/statistics

3. UPDATE
   - Refresh introduction and conclusion
   - Add new sections/subsections
   - Update statistics and examples
   - Improve internal linking
   - Optimize for new keywords

4. REPUBLISH
   - Update publish date (if significant changes)
   - Reshare on social media
   - Notify email subscribers
   - Build new backlinks

5. MONITOR
   - Track ranking changes (2-4 weeks)
   - Monitor traffic recovery
   - Measure engagement improvements
```

## 7. Link Building Integration

### 7.1 Internal Linking Strategy

**Hub & Spoke Model:**
```
         PILLAR PAGE (Broad Topic)
              │
    ┌─────────┼─────────┐
    │         │         │
 CLUSTER   CLUSTER   CLUSTER
  PAGE 1    PAGE 2    PAGE 3
    │         │         │
    └─────────┴─────────┘
         (Interlinked)
```

**Internal Link Rules:**
- Link from high-authority pages to new content
- Use descriptive anchor text (not "click here")
- Link deep (not just to homepage)
- Maintain logical topic clusters
- Update old content with links to new content

### 7.2 External Link Guidelines

| Do | Don't |
|----|-------|
| Link to authoritative sources | Link to direct competitors |
| Use relevant, contextual links | Over-optimize anchor text |
| Open external links in new tab | Link to low-quality sites |
| Cite original research/data | Break links (check regularly) |
