# Production Process

End-to-end workflow for creating, editing, optimizing, and publishing content at scale.

---

## Overview

Our production process is designed for:
- **Quality** - Consistent, high-standard output
- **Efficiency** - Scalable workflows
- **Clarity** - Clear ownership at each stage
- **Flexibility** - Adaptable to different content types

```
BRIEFING → CREATION → EDITING → SEO → APPROVAL → PUBLISHING → DISTRIBUTION
   ↓          ↓          ↓        ↓         ↓           ↓            ↓
  1 day    2-5 days    1-2 days  0.5 day   1-2 days    0.5 day     Ongoing
```

---

## Stage 1: Briefing

**Goal**: Provide creators with everything needed to produce great content.

### The Brief

Every piece of content starts with a detailed brief. No exceptions.

**Why Briefs Matter:**
- Eliminates guesswork
- Reduces revision rounds
- Ensures strategic alignment
- Enables parallel work streams

### Brief Components

```markdown
# Content Brief: [Title]

## 1. Overview
- **Content Type**: Blog post / Whitepaper / Video / etc.
- **Working Title**: 
- **Target Keyword**: 
- **Word Count/Duration**: 
- **Due Date**: 
- **Assigned To**: 

## 2. Objective
- **Primary Goal**: (traffic, leads, awareness, etc.)
- **Funnel Stage**: (TOFU, MOFU, BOFU)
- **Desired Action**: (what should reader do after consuming?)

## 3. Audience
- **Primary Persona**: 
- **Knowledge Level**: (beginner, intermediate, advanced)
- **Key Question**: (what question does this answer?)
- **Objections to Address**: 

## 4. SEO Requirements
- **Primary Keyword**: 
- **Secondary Keywords**: (2-3)
- **Search Intent**: (informational, commercial, transactional)
- **Competitor URLs**: (top 3 ranking pages)
- **Internal Links**: (3-5 URLs to include)

## 5. Outline
### H1: [Title]
### Introduction (100-150 words)
- Hook approach
- Context to provide
- Promise to reader

### H2: [Section 1]
- Key points to cover
- Data/sources needed
- Examples to include

### H2: [Section 2]
...

### Conclusion
- Key takeaways to summarize
- CTA details

## 6. Brand Voice
- **Tone**: (professional, conversational, authoritative, etc.)
- **Style Guide**: [Link]
- **Words to Use**: 
- **Words to Avoid**: 
- **Example Content**: [Links to 2-3 reference pieces]

## 7. Assets Needed
- [ ] Custom graphics
- [ ] Screenshots
- [ ] Data visualization
- [ ] Video clips
- [ ] Expert quotes

## 8. Additional Notes
- Special requirements
- Compliance considerations
- Stakeholder sensitivities
```

### Brief Creation Process

| Step | Owner | Time |
|------|-------|------|
| Topic selection | Strategy Lead | 30 min |
| Keyword research | SEO Specialist | 30 min |
| Outline development | Strategy Lead | 1 hour |
| Brief completion | Content Manager | 30 min |
| Brief approval | Client/Lead | 1 day |

### Brief Quality Checklist

- [ ] Clear objective stated
- [ ] Target persona identified
- [ ] Keyword research complete
- [ ] Outline is logical and complete
- [ ] Internal links identified
- [ ] Brand voice guidance included
- [ ] Examples provided
- [ ] Due date is realistic

## Stage 2: Creation

**Goal**: Produce first draft that meets brief requirements.

### Writer Guidelines

**Before Writing:**
1. Read the brief thoroughly
2. Review example content
3. Research the topic (beyond provided sources)
4. Create outline if not provided
5. Gather all necessary sources

**During Writing:**
1. Follow the brief outline
2. Write for the target persona
3. Keep SEO requirements in mind
4. Flag any issues early (don't wait)
5. Meet the word count target (±10%)

**After Writing:**
1. Self-edit for clarity
2. Check all links work
3. Verify facts and sources
4. Ensure CTA is clear
5. Submit on time

### Content Creation Standards

**Written Content:**

| Element | Standard |
|---------|----------|
| Readability | Grade 8-10 (Hemingway) |
| Sentence Length | Avg. 15-20 words |
| Paragraph Length | 2-4 sentences |
| Active Voice | 80%+ active |
| Transition Words | 30%+ of sentences |

**Video Content:**

| Element | Standard |
|---------|----------|
| Hook | First 5 seconds |
| Pacing | Cut every 3-5 seconds |
| Audio | Clear, no background noise |
| Lighting | Professional, consistent |
| Branding | Intro/outro, lower thirds |

**Audio/Podcast:**

| Element | Standard |
|---------|----------|
| Audio Quality | -16 LUFS, noise floor -60dB |
| Intro | 30-60 seconds max |
| Segments | Clear transitions |
| Outro | CTA + next episode tease |

## Stage 3: Editing

**Goal**: Elevate content quality through systematic review.

### Three-Tier Editing Process

```
┌─────────────────────────────────────────────────────────┐
│  TIER 1: DEVELOPMENTAL EDIT                             │
│  - Structure and flow                                   │
│  - Argument strength                                    │
│  - Completeness                                         │
│  - Strategic alignment                                  │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│  TIER 2: LINE EDIT                                      │
│  - Clarity and readability                              │
│  - Voice and tone                                       │
│  - Transitions                                          │
│  - Engagement                                           │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│  TIER 3: COPY EDIT                                      │
│  - Grammar and spelling                                 │
│  - Punctuation                                          │
│  - Consistency                                          │
│  - Fact-checking                                        │
└─────────────────────────────────────────────────────────┘
```

### Developmental Edit Checklist

- [ ] Structure follows the brief
- [ ] Introduction hooks effectively
- [ ] Logical flow between sections
- [ ] Each section supports the main point
- [ ] Examples and data support claims
- [ ] Conclusion summarizes key points
- [ ] CTA is clear and appropriate
- [ ] Content meets strategic objective

### Line Edit Checklist

- [ ] Sentences are clear and concise
- [ ] Paragraphs are scannable
- [ ] Voice matches brand guidelines
- [ ] Transitions are smooth
- [ ] Jargon is explained or removed
- [ ] Reading level is appropriate
- [ ] Content is engaging throughout

### Copy Edit Checklist

- [ ] No spelling errors
- [ ] Grammar is correct
- [ ] Punctuation is correct
- [ ] Consistent style (AP, Chicago, house)
- [ ] Consistent formatting
- [ ] Links work and are formatted correctly
- [ ] Facts are accurate
- [ ] Sources are cited properly

### Editing Timeline

| Content Type | Dev Edit | Line Edit | Copy Edit | Total |
|--------------|----------|-----------|-----------|-------|
| Blog Post | 30 min | 30 min | 20 min | 1.5 hours |
| Whitepaper | 2 hours | 2 hours | 1 hour | 5 hours |
| Case Study | 1 hour | 1 hour | 30 min | 2.5 hours |
| Ebook | 4 hours | 4 hours | 2 hours | 10 hours |

### Editor Feedback Format

Use clear, actionable feedback:

**Good:**
> "This section feels long. Consider breaking into two H3s: 'Challenge 1' and 'Challenge 2'. Also, add a stat to support the claim in paragraph 2."

**Bad:**
> "This section needs work."

## Stage 4: SEO Optimization

**Goal**: Ensure content is optimized for search engines without sacrificing quality.

### SEO Checklist

**On-Page Elements:**
- [ ] Primary keyword in title tag (front-loaded)
- [ ] Title tag under 60 characters
- [ ] Meta description under 155 characters
- [ ] Meta description includes keyword and CTA
- [ ] URL slug is clean and includes keyword
- [ ] H1 includes primary keyword
- [ ] Primary keyword in first 100 words
- [ ] Primary keyword in at least one H2
- [ ] Secondary keywords naturally integrated
- [ ] Keyword density 1-2% (natural, not forced)

**Content Quality:**
- [ ] Content is comprehensive (covers topic fully)
- [ ] Content is original (not thin or duplicate)
- [ ] Readability score is appropriate
- [ ] Internal links included (3-5)
- [ ] External links to authoritative sources (2-5)
- [ ] Images have descriptive alt text
- [ ] Content is scannable (headers, bullets, short paragraphs)

**Technical:**
- [ ] Schema markup added (if applicable)
- [ ] Page load speed is optimized
- [ ] Mobile-friendly formatting
- [ ] No broken links

### SEO Tools

| Tool | Purpose |
|------|---------|
| Ahrefs/SEMrush | Keyword research, competitor analysis |
| Surfer SEO | Content optimization |
| Clearscope | Content grading |
| Yoast/RankMath | WordPress optimization |
| Google Search Console | Performance tracking |
| PageSpeed Insights | Speed optimization |

### SEO Review Process

1. **Automated Check** - Run through SEO tool (Surfer, Clearscope)
2. **Manual Review** - SEO specialist reviews content
3. **Recommendations** - Document required changes
4. **Implementation** - Writer or SEO specialist makes changes
5. **Final Check** - Confirm all optimizations in place

**Time**: 30-60 minutes per blog post

## Stage 5: Approval

**Goal**: Get stakeholder sign-off before publishing.

### Approval Workflow

```
Writer → Editor → SEO → Client/Stakeholder → Publishing
   ↓        ↓       ↓          ↓              ↓
 Draft   Quality  Search    Final OK      Go Live
```

### Approval Process

| Step | Owner | Time Allowed | Action |
|------|-------|--------------|--------|
| Internal Review | Content Manager | 1 day | Quality check |
| SEO Review | SEO Specialist | 0.5 day | Optimization check |
| Client Review | Client | 2-3 days | Final approval |
| Revisions | Writer | 1-2 days | Implement feedback |
| Final Approval | Client | 1 day | Sign-off |

### Client Review Guidelines

**What to Provide:**
- Google Doc with comment access
- Clear deadline for feedback
- Specific questions if needed
- Contact for urgent questions

**What to Expect:**
- 2 revision rounds included
- Feedback within 3 business days
- Clear rationale for changes
- Final approval before publishing

### Common Approval Issues

| Issue | Prevention |
|-------|------------|
| Vague feedback | Ask for specific examples |
| Endless revisions | Set clear round limits |
| Late feedback | Build buffer into timeline |
| Stakeholder conflicts | Identify single decision-maker |
| Scope creep | Reference original brief |

## Stage 6: Publishing

**Goal**: Publish content correctly and consistently.

### Publishing Checklist

**Website/Blog:**
- [ ] Title tag optimized
- [ ] Meta description added
- [ ] URL slug correct
- [ ] H1 matches title
- [ ] Headers formatted correctly (H2, H3)
- [ ] Images optimized (compressed, alt text)
- [ ] Internal links added
- [ ] External links added
- [ ] CTA buttons/links working
- [ ] Schema markup added (if applicable)
- [ ] Categories/tags assigned
- [ ] Featured image set
- [ ] Preview checked (desktop + mobile)
- [ ] Publish date/time set
- [ ] Published

**Video:**
- [ ] Title optimized
- [ ] Description complete
- [ ] Tags added
- [ ] Thumbnail uploaded
- [ ] Captions added
- [ ] End screens/cards added
- [ ] Playlist assigned
- [ ] Published

**Podcast:**
- [ ] Episode title set
- [ ] Show notes complete
- [ ] Audio file uploaded
- [ ] Episode artwork set
- [ ] Timestamps added
- [ ] Links included
- [ ] Published to all platforms

### CMS Guidelines

**WordPress:**
- Use Yoast or RankMath for SEO
- Follow heading hierarchy
- Optimize images before upload
- Use categories and tags consistently
- Set featured image (1200x630px)

**Other Platforms:**
- Follow platform-specific best practices
- Maintain consistent formatting
- Test before publishing

## Stage 7: Distribution

**Goal**: Maximize content reach and impact.

### Distribution Checklist (Per Piece)

**Day 0 (Publish Day):**
- [ ] Publish on website
- [ ] Send email newsletter
- [ ] Post on LinkedIn (company + personal)
- [ ] Post on Twitter/X
- [ ] Post on other relevant social platforms
- [ ] Share in relevant communities (if allowed)
- [ ] Notify sales team

**Day 1-3:**
- [ ] Share variation on social
- [ ] Respond to comments
- [ ] Share in internal channels
- [ ] Add to relevant resource pages

**Day 7:**
- [ ] Share "best of" roundup
- [ ] Consider paid promotion
- [ ] Repurpose into different format

**Ongoing:**
- [ ] Include in relevant emails
- [ ] Share when contextually relevant
- [ ] Update and reshare (quarterly)

### Distribution by Channel

| Channel | Format | Frequency | Owner |
|---------|--------|-----------|-------|
| Email Newsletter | Link + summary | Next send | Email Manager |
| LinkedIn | Post + comments | Day 0, 3, 7 | Social Manager |

| Instagram | Graphic + link in bio | Day 0, 4 | Social Manager |
