## 1. Lead Management

### 1.1 Lead Capture

**Sources:**
- Website contact form
- LinkedIn outreach responses
- Referral introductions
- Event/conference connections
- Cold email responses

**Action:** Enter all leads into CRM within 24 hours.

### 1.2 Lead Qualification (BANT)

**Budget:** Do they have budget for $5K+ projects?
**Authority:** Are they the decision-maker?
**Need:** Do they have a clear pain point we can solve?
**Timeline:** Are they looking to solve this in the next 90 days?

**Scoring:**
- ✅✅✅✅ = Hot (prioritize)
- ✅✅✅ = Warm (nurture)
- ✅✅ or less = Cold (long-term nurture or disqualify)

### 1.3 Lead Response Time

| Lead Source | Response Target |
|-------------|-----------------|
| Website form | Within 2 hours |
| Referral | Within 1 hour |
| LinkedIn | Within 4 hours |
| Cold email reply | Within 2 hours |
| Event connection | Within 24 hours |

### 1.4 CRM Pipeline Stages

```
New Lead → Contacted → Discovery Scheduled → Discovery Complete → 
Proposal Sent → Negotiation → Closed Won / Closed Lost
```

## 2. Discovery Call Process

### 2.1 Pre-Call Prep (15 minutes)

**Checklist:**
- [ ] Research company website
- [ ] Check LinkedIn profiles of attendees
- [ ] Google recent news about company
- [ ] Review any previous communications
- [ ] Prepare 5-10 discovery questions
- [ ] Test audio/video setup
- [ ] Open CRM for note-taking

### 2.2 During Call (30-45 minutes)

**Structure:**
1. **Opening (5 min):** Build rapport, set expectations
2. **Discovery (20 min):** Ask questions, listen actively
3. **Solution Overview (10 min):** Present 2-3 options
4. **Next Steps (5 min):** Agree on follow-up

**Note-Taking Template:**
```
Pain Points:
- 
- 
- 

Current Process:
- 

Impact (time/money):
- 

Decision Makers:
- 

Timeline:
- 

Budget Range:
- 

Recommended Solution:
- 

Follow-Up Actions:
- 
```

### 2.3 Post-Call Actions (Within 1 Hour)

- [ ] Update CRM with call notes
- [ ] Tag lead with appropriate status
- [ ] Schedule follow-up task
- [ ] Send thank-you email
- [ ] If qualified: Begin proposal draft

## 3. Proposal Creation

### 3.1 Proposal Timeline

**Standard:** Send within 24-48 hours of discovery call
**Rush:** Same day (for hot leads)

### 3.2 Proposal Checklist

- [ ] Use proposal template
- [ ] Customize executive summary
- [ ] Include client's pain points (in their words)
- [ ] Present 2-3 solution options
- [ ] Include ROI calculation
- [ ] Add relevant case studies
- [ ] Review pricing against guidelines
- [ ] Proofread for errors
- [ ] Export as PDF
- [ ] Send via email with tracking

### 3.3 Proposal Follow-Up Sequence

| Day | Action |
|-----|--------|
| 0 | Send proposal |
| 3 | Check-in email |
| 7 | Value-add follow-up |
| 14 | Direct ask |
| 30 | Break-up email |

## 4. Client Onboarding

### 4.1 Upon Contract Signing

**Within 24 Hours:**
- [ ] Send welcome email
- [ ] Schedule kickoff call
- [ ] Send onboarding questionnaire
- [ ] Create project in project management tool
- [ ] Set up client folder in Google Drive
- [ ] Add client to CRM as "Active"

**Welcome Email Template:**
```
Subject: Welcome to [Agency Name]! 🎉

Hey [Name],

We're excited to work with you on [project]!

Here's what happens next:
1. Fill out the onboarding questionnaire [link]
2. Schedule our kickoff call [Calendly link]
3. We'll send access requests for [tools]

If you have any questions before we kick off, just reply to this email!

Best,
[Your Name]
```

### 4.2 Onboarding Questionnaire

**Collect:**
- Company overview and org chart
- Key stakeholders and decision-makers
- Current tools and tech stack
- Access requirements (systems, APIs, etc.)
- Brand guidelines (if applicable)
- Communication preferences
- Success metrics and KPIs

### 4.3 Access Management

**Create a secure document with:**
- System names and URLs
- Contact persons for access requests
- Access status (requested/granted)
- Login credentials (use password manager)

## 5. Project Kickoff

### 5.1 Kickoff Meeting Agenda (30-45 minutes)

**Attendees:**
- Client stakeholders
- Your project team
- Key contractors (if applicable)

**Agenda:**
1. **Introductions (5 min)**
2. **Project Overview (10 min)**
   - Goals and success metrics
   - Scope review
   - Timeline and milestones
3. **Process & Communication (10 min)**
   - Weekly check-in schedule
   - Communication channels
   - Decision-making process
4. **Next Steps (5 min)**
   - Immediate action items
   - Deadlines
5. **Q&A (10-15 min)**

### 5.2 Kickoff Deliverables

- [ ] Meeting recording (Loom/Zoom)
- [ ] Meeting notes sent within 24 hours
- [ ] Project plan shared
- [ ] Communication channels set up
- [ ] First milestone tasks assigned

### 5.3 Project Setup Checklist

- [ ] Create project in ClickUp/Asana
- [ ] Set up Slack channel (internal + client)
- [ ] Create shared Google Drive folder
- [ ] Add all team members
- [ ] Upload SOW and contract
- [ ] Set up recurring check-in meetings
- [ ] Create status report template

## 6. Contractor Management

### 6.1 Contractor Sourcing

**When to Hire:**
- New project requires specific skill
- Current capacity exceeded
- Need specialized expertise

**Where to Look:**
1. Existing contractor network (first choice)
2. Referrals from other contractors
3. Upwork (Top Rated Plus)
4. Toptal (for senior roles)
5. LinkedIn
6. Twitter/X communities
7. Discord communities

### 6.2 Contractor Vetting Process

**Step 1: Initial Screening (30 min)**
- [ ] Review portfolio
- [ ] Check rates vs. budget
- [ ] Confirm availability
- [ ] Initial video chat

**Step 2: Technical Assessment (1 hour)**
- [ ] Skills verification
- [ ] Problem-solving exercise
- [ ] Communication evaluation

**Step 3: Trial Project ($500-1,000)**
- [ ] Small paid test project
- [ ] Evaluate quality
- [ ] Evaluate timeliness
- [ ] Evaluate communication

**Step 4: Onboarding**
- [ ] Sign NDA and contractor agreement
- [ ] Add to contractor roster
- [ ] Grant tool access
- [ ] Process training

### 6.3 Contractor Brief Template

```
Project: [Name]
Role: [Role]
Timeline: [Dates]
Rate: $[X]/hr
Estimated Hours: [X]

Overview:
[Brief description]

Deliverables:
- [Specific deliverable 1]
- [Specific deliverable 2]

Requirements:
- [Technical requirements]
- [Quality standards]

Access:
- [Tools/systems they'll need]

Deadlines:
- [Milestone 1]: [Date]
- [Final]: [Date]

Questions? Reach out to [Your Name] at [contact info]
```

### 6.4 Contractor Payment

**Process:**
1. Contractor submits timesheet/invoice weekly
2. Review hours and work completed
3. Approve in system
4. Payment processed bi-weekly (Fridays)
5. Confirm receipt

**Payment Terms:** Net 7 or Net 15

## 7. Quality Assurance

### 7.1 QA Checklist (All Projects)

**Before Client Delivery:**
- [ ] All requirements met
- [ ] Functionality tested
- [ ] Edge cases handled
- [ ] Error messages clear
- [ ] Performance acceptable
- [ ] Documentation complete
- [ ] Code reviewed (if applicable)
- [ ] Security considerations addressed

### 7.2 QA by Project Type

**AI Agents:**
- [ ] Test with 50+ varied inputs
- [ ] Verify accuracy >90%
- [ ] Test escalation paths
- [ ] Verify integrations working
- [ ] Test error handling

**Workflow Automations:**
- [ ] Test full workflow end-to-end
- [ ] Test error scenarios
- [ ] Verify all integrations
- [ ] Check logging is working
- [ ] Test recovery procedures

**Custom Tools:**
- [ ] User acceptance testing
- [ ] Load testing (if applicable)
- [ ] Security review
- [ ] Cross-browser/device testing
- [ ] API documentation complete

### 7.3 Client UAT Process

1. **Prepare UAT Environment**
   - Set up staging/test environment
   - Create test data
   - Prepare UAT instructions

2. **Send UAT Package**
   - Access credentials
   - Test scenarios
   - Expected results
   - Feedback form

3. **Review Feedback**
   - Categorize: Bug vs. Enhancement
   - Prioritize fixes
   - Communicate timeline

4. **Get Sign-Off**
   - Written UAT approval
   - Document any known issues
   - Plan for post-launch fixes

## 9. Client Offboarding

### 9.1 When to Offboard

- Project completed, no retainer
- Client requests termination
- Contract ends, not renewed
- Non-payment (after collections)

### 9.2 Offboarding Checklist

- [ ] Final deliverables transferred
- [ ] All access credentials returned
- [ ] Documentation handed off
- [ ] Final invoice sent
- [ ] Access revoked (our systems)
- [ ] Project archived
- [ ] Feedback survey sent
- [ ] Testimonial requested (if positive)
- [ ] CRM status updated
- [ ] Team notified

### 9.3 Offboarding Email Template

```
Subject: Project Wrap-Up & Next Steps

Hey [Name],

As we wrap up [project], I wanted to make sure you have everything you need:

✅ All deliverables: [Link]
✅ Documentation: [Link]
✅ Access credentials: [Sent separately]
✅ Support contact: [Email/phone]

Your [30-day] support period runs through [date]. After that, 
we'd love to continue supporting you via our retainer program.

It's been a pleasure working with you! If you need anything 
in the future, don't hesitate to reach out.

Best,
[Your Name]
```

## 11. Financial Operations

### 11.1 Invoicing Process

**When to Invoice:**
- Deposit: Upon contract signing
- Milestones: Upon milestone completion
- Final: Upon project delivery
- Retainers: 1st of each month

**Invoice Checklist:**
- [ ] Correct client details
- [ ] Clear description of services
- [ ] Correct amount
- [ ] Payment terms stated
- [ ] Payment methods listed
- [ ] Due date clear

### 11.2 Payment Tracking

**Weekly Review:**
- [ ] Check outstanding invoices
- [ ] Follow up on invoices >7 days overdue
- [ ] Update CRM payment status
- [ ] Record payments received

**Collections Process:**
| Days Overdue | Action |
|--------------|--------|
| 1-7 | Friendly reminder email |
| 8-15 | Direct email + phone call |
| 16-30 | Formal notice, work pause |
| 31-60 | Collections agency consideration |
| 60+ | Legal action |

### 11.3 Financial Reporting

**Monthly:**
- Revenue by service type
- Revenue by client
- Accounts receivable aging
- Contractor costs
- Profit margin per project
- Cash flow projection

**Quarterly:**
- P&L statement
- Tax estimates
- Budget vs. actual
- Growth metrics
