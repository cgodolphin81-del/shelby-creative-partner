# Discovery Workshop Checklist

## Pre-Workshop Preparation

### 1-2 Weeks Before
- [ ] Send calendar invites to all attendees
- [ ] Share pre-workshop questionnaire
- [ ] Review client website and materials
- [ ] Research industry and competitors
- [ ] Prepare workshop agenda
- [ ] Set up Miro/Mural board
- [ ] Test video conferencing setup

### 1 Day Before
- [ ] Confirm attendance
- [ ] Send reminder with agenda
- [ ] Review pre-workshop responses
- [ ] Prepare introductions
- [ ] Test screen sharing
- [ ] Print materials (if in-person)

---

## Workshop Agenda

### Morning Session (3-4 hours)

#### 1. Introductions & Context (30 min)
- [ ] Welcome and housekeeping
- [ ] Participant introductions
- [ ] Workshop objectives
- [ ] Agenda review
- [ ] Ground rules

#### 2. Business Goals & Objectives (60 min)
- [ ] Company overview
- [ ] Strategic priorities (this year)
- [ ] Department goals
- [ ] Success metrics
- [ ] Key challenges

**Questions to Ask:**
- What are your top 3 business priorities this year?
- What does success look like in 12 months?
- What's keeping you up at night?
- How do you currently measure success?

#### 3. Current State Assessment (60 min)
- [ ] Current tools and systems
- [ ] Data sources overview
- [ ] Reporting processes
- [ ] Pain points
- [ ] What's working well

**Questions to Ask:**
- What tools are you using today?
- Where does your data live?
- How do you create reports currently?
- What takes too long?
- What frustrates your team?

#### 4. Pain Points & Challenges (45 min)
- [ ] Identify top 5 pain points
- [ ] Prioritize by impact
- [ ] Discuss root causes
- [ ] Document examples

**Activity:** Pain point prioritization matrix
- High impact / Easy to fix
- High impact / Hard to fix
- Low impact / Easy to fix
- Low impact / Hard to fix

#### 5. Success Criteria Definition (45 min)
- [ ] Define success metrics
- [ ] Set targets
- [ ] Identify quick wins
- [ ] Document must-haves

**Questions to Ask:**
- What would make this project a home run?
- What's the minimum viable outcome?
- How will you measure success?
- What's the cost of inaction?

---

### Afternoon Session (2-4 hours)

#### 6. Data Source Mapping (60 min)
- [ ] List all data sources
- [ ] Document owners
- [ ] Assess quality
- [ ] Identify access method
- [ ] Note integration complexity

**Template:**
| Source | System | Owner | Update Freq | Quality | Access |
|--------|--------|-------|-------------|---------|--------|
| | | | | | |

#### 7. Technical Deep Dive (60 min)
- [ ] IT infrastructure overview
- [ ] Security requirements
- [ ] Compliance needs
- [ ] Existing integrations
- [ ] Technical constraints

**Questions to Ask:**
- What's your tech stack?
- Any security/compliance requirements?
- What systems must we integrate with?
- Any technical constraints we should know?

#### 8. User Needs & Personas (45 min)
- [ ] Identify user groups
- [ ] Define personas
- [ ] Document use cases
- [ ] Prioritize by importance

**Personas to Consider:**
- Executive (C-suite, VP)
- Manager (Director, Manager)
- Analyst (Data analyst, Business analyst)
- Operational (Front-line, individual contributor)

#### 9. Prioritization Exercise (45 min)
- [ ] List all requirements
- [ ] Vote on priorities
- [ ] Create must/should/could have
- [ ] Agree on Phase 1 scope

**Activity:** MoSCoW prioritization
- Must have (non-negotiable)
- Should have (important but not vital)
- Could have (nice to have)
- Won't have (not now)

#### 10. Next Steps & Timeline (30 min)
- [ ] Recap key findings
- [ ] Outline next steps
- [ ] Discuss timeline
- [ ] Assign action items
- [ ] Schedule follow-up

---

## Post-Workshop Activities

### Within 24 Hours
- [ ] Send thank you email
- [ ] Share workshop notes
- [ ] Distribute recording (if applicable)
- [ ] Request any missing information

### Within 3-5 Days
- [ ] Complete data audit
- [ ] Document findings
- [ ] Create architecture draft
- [ ] Begin proposal development

### Within 1 Week
- [ ] Send proposal
- [ ] Schedule proposal review
- [ ] Address questions
- [ ] Move toward close

## Success Criteria

### Workshop is Successful When:
- [ ] All key stakeholders attended
- [ ] Pain points clearly identified
- [ ] Success criteria defined
- [ ] Data sources mapped
- [ ] Priorities agreed upon
- [ ] Next steps clear
- [ ] Client feels heard and understood

### Red Flags
- [ ] Key decision maker absent
- [ ] Unclear about goals
- [ ] Unwilling to share information
- [ ] Unrealistic expectations
- [ ] No budget identified
- [ ] Timeline impossible

## Common Pitfalls & How to Avoid

| Pitfall | Solution |
|---------|----------|
| Too many attendees | Limit to 8-10 key people |
| Dominating personalities | Use structured activities |
| Going off-topic | Park issues in parking lot |
| Technical deep-dives | Schedule separate session |
| No decision maker present | Reschedule or get commitment |
| Vague requirements | Use specific examples |
| Unrealistic timeline | Educate on typical timelines |

---

## Facilitator Tips

### Do's
✅ Start on time, end on time
✅ Encourage equal participation
✅ Summarize key points frequently
✅ Use visual collaboration tools
✅ Take detailed notes
✅ Build rapport
✅ Listen more than talk

### Don'ts
❌ Don't promise specific solutions
❌ Don't criticize current state
❌ Don't get bogged down in details
❌ Don't skip the prioritization
❌ Don't leave without next steps
❌ Don't oversell
❌ Don't talk over clients
