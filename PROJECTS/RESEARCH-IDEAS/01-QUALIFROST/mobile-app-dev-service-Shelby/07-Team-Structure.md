# 07. Team Structure

Organizational design, roles, hiring guides, and team scaling strategies.

---

## Team Structure Overview

### Agency Stages

```
STAGE 1: Founder-Led (0-6 months)
├─ Founder (sales + PM + some dev)
└─ 1-2 Contractors (development)
Revenue: $0-500K

STAGE 2: Core Team (6-18 months)
├─ Founder (sales + strategy)
├─ 1 Project Manager
├─ 2-3 Developers
├─ 1 Designer (FT or contractor)
└─ Contractors as needed
Revenue: $500K-1.5M

STAGE 3: Growing Agency (18-36 months)
├─ Founder/CEO
├─ 1 Account Director
├─ 2 Project Managers
├─ 4-6 Developers
├─ 2 Designers
├─ 1 QA Engineer
└─ 1 DevOps (or contractor)
Revenue: $1.5M-3M

STAGE 4: Established Agency (36+ months)
├─ CEO
├─ Head of Delivery
├─ Head of Sales
├─ 3-4 Project Managers
├─ 8-12 Developers
├─ 3-4 Designers
├─ 2-3 QA Engineers
├─ 1-2 DevOps
└─ Marketing/Operations support
Revenue: $3M-10M+
```

## Core Roles

### 1. Founder/CEO

**Responsibilities:**
- Vision and strategy
- Sales and business development
- Key client relationships
- Hiring and culture
- Financial management
- Final decision maker

**Skills Required:**
- Sales and negotiation
- Strategic thinking
- Industry knowledge
- Leadership
- Financial acumen

**Time Allocation (Stage 1-2):**
- 40% Sales
- 20% Client management
- 20% Hiring/team
- 10% Strategy
- 10% Operations

**When to Hire:** You are the first CEO. Consider COO at $2M+ revenue.

---

### 2. Project Manager (PM)

**Responsibilities:**
- Client communication (day-to-day)
- Sprint planning and facilitation
- Timeline and budget tracking
- Risk management
- Resource coordination
- Status reporting

**Skills Required:**
- Agile/Scrum expertise
- Communication skills
- Organization
- Problem-solving
- Basic technical understanding

**Experience Level:**
- Junior PM: 2-3 years, $60-80K
- Mid PM: 4-6 years, $80-110K
- Senior PM: 7+ years, $110-140K

**Ratio:** 1 PM per 2-3 development teams (or $500K-750K revenue)

**Interview Questions:**
1. Walk me through how you'd handle a project that's falling behind.
2. How do you manage difficult client conversations?
3. Describe your experience with Agile/Scrum.
4. What tools do you use for project management?
5. Tell me about a project that failed. What happened?

---

### 3. Mobile Developer

#### Junior Developer
**Experience:** 1-3 years
**Rate:** $60-90K or $50-75/hr contract

**Can Do:**
- Implement defined features
- Fix bugs
- Write basic tests
- Follow established patterns

**Needs Support:**
- Architecture decisions
- Complex integrations
- Performance optimization
- Code review guidance

#### Mid-Level Developer
**Experience:** 3-6 years
**Rate:** $90-130K or $75-125/hr contract

**Can Do:**
- Own features end-to-end
- Make architectural contributions
- Mentor juniors
- Client communication
- Estimate work accurately

**Needs Support:**
- Complex system design
- Cross-team coordination
- Strategic decisions

#### Senior Developer
**Experience:** 6+ years
**Rate:** $130-180K or $125-200/hr contract

**Can Do:**
- System architecture
- Technical leadership
- Client technical discussions
- Code review and quality
- Performance optimization
- Team mentoring

**Ideal For:**
- Tech lead roles
- Complex projects
- Client-facing technical work

#### Tech Lead
**Experience:** 8+ years
**Rate:** $150-220K

**Responsibilities:**
- Technical architecture
- Code quality standards
- Team mentoring
- Client technical advisory
- Technology decisions
- Hiring technical team

**Ratio:** 1 Tech Lead per 4-6 developers

## Hiring Process

### Sourcing Channels

| Channel | Best For | Cost | Quality |
|---------|----------|------|---------|
| LinkedIn | All roles | $0-500/post | High |
| Referrals | All roles | $2-5K bonus | Very High |
| Indeed/Glassdoor | Junior-Mid | $200-500/post | Medium |
| AngelList | Startups-minded | $200-400/post | High |
| Toptal/Gun.io | Senior/Contract | 20-30% markup | Very High |
| Local meetups | All roles | $50-200/event | Medium-High |
| Universities | Junior | $0-500 | Medium |

### Interview Process

**Standard Process (5 steps):**

```
Application → Screen (30 min) → Technical (60 min) → 
Culture Fit (45 min) → Reference Check → Offer
```

**Timeline:** 2-3 weeks from application to offer

#### Step 1: Application Review
**Look for:**
- Relevant experience
- Portfolio/GitHub
- Clear communication
- Career progression

**Red flags:**
- Job hopping (<1 year per role, repeated)
- Gaps without explanation
- Typos/poor communication
- Generic applications

#### Step 2: Phone Screen (30 min)
**Agenda:**
- Background and experience (10 min)
- Role interest and motivation (10 min)
- Salary expectations and logistics (5 min)
- Candidate questions (5 min)

**Pass criteria:**
- Clear communication
- Genuine interest
- Salary alignment
- No red flags

#### Step 3: Technical Assessment

**For Developers:**

**Option A: Take-home (3-5 hours)**
```
Build a simple app with:
- User authentication
- API integration
- List/detail views
- Basic state management

Evaluation:
- Code quality
- Architecture
- Testing
- Completeness
```

**Option B: Live coding (60 min)**
```
Pair programming session:
- Build a small feature
- Discuss approach
- Problem-solving

Evaluation:
- Problem-solving
- Communication
- Technical knowledge
- Collaboration
```

**Option C: Code review (45 min)**
```
Review provided code:
- Identify issues
- Suggest improvements
- Discuss trade-offs

Evaluation:
- Code quality eye
- Communication
- Technical depth
```

**For Designers:**
- Portfolio review (deep dive)
- Design exercise (take-home or whiteboard)
- Tool proficiency check

**For PMs:**
- Case study discussion
- Scenario-based questions
- Tool familiarity

#### Step 4: Culture Fit Interview (45 min)
**Interviewers:** Founder + potential teammate

**Questions:**
1. Tell me about a time you disagreed with a teammate. How did you handle it?
2. What's your ideal work environment?
3. Describe a project you're proud of and why.
4. How do you handle tight deadlines?
5. What are you looking for in your next role?

**Evaluate:**
- Values alignment
- Communication style
- Growth mindset
- Team fit

#### Step 5: Reference Checks
**Questions for references:**
1. How would you describe working with [candidate]?
2. What are their greatest strengths?
3. What areas did they improve in during their time with you?
4. Would you rehire them? Why or why not?
5. Is there anything else I should know?

## Team Scaling Strategy

### Revenue-Based Hiring Triggers

| Revenue Milestone | Hire |
|-------------------|------|
| $250K | First contractor developer |
| $500K | First full-time PM |
| $750K | Second developer |
| $1M | First designer (FT), third developer |
| $1.5M | First QA, fourth developer |
| $2M | First sales hire, marketing |
| $3M | Second PM, DevOps |
| $5M | Department heads, expand teams |

### Utilization Targets

| Role | Target Utilization |
|------|-------------------|
| Developers | 80-85% (32-34 hrs/week billable) |
| Designers | 75-80% |
| PMs | 90%+ (mostly billable) |
| QA | 80-85% |

**Calculation:**
```
Utilization = Billable Hours / Available Hours

Example:
Developer available: 40 hrs/week
Billable target: 32 hrs/week
Utilization: 32/40 = 80%

Non-billable time:
- Internal meetings
- Training
- Bench time
- Admin
```

### Bench Management

**Ideal bench:** 10-15% of team capacity

**When bench is high (>20%):**
- Pause hiring
- Focus on sales
- Invest in training
- Consider contractors over FT

**When bench is low (<5%):**
- Start recruiting now (hiring takes 6-10 weeks)
- Consider contractors for overflow
- Evaluate pricing (may be underpriced)

## Remote vs. In-Office

### Remote-First (Recommended)

**Pros:**
- Larger talent pool
- Lower overhead
- Flexible for team
- Competitive advantage

**Cons:**
- Culture building harder
- Communication challenges
- Time zone coordination

**Best Practices:**
- Core overlap hours (4+ hours)
- Regular video check-ins
- Annual in-person gatherings
- Documentation-first culture
- Async communication norms

### Hybrid

**Structure:**
- 2-3 days in office
- Flexible remote days
- Local team only

**Best For:**
- Teams in major tech hubs
- Culture-focused companies
- Roles requiring collaboration

## Performance Management

### Review Cadence

- **Weekly:** 1:1s with direct reports
- **Monthly:** Performance check-ins
- **Quarterly:** Formal reviews
- **Annually:** Compensation review

### 1:1 Template (30 min)

| Time | Topic |
|------|-------|
| 0-5 min | Personal check-in |
| 5-15 min | Current work, blockers |
| 15-20 min | Feedback (both directions) |
| 20-25 min | Growth and development |
| 25-30 min | Action items |

### Performance Review Criteria

**For Developers:**
- Code quality
- Delivery reliability
- Problem-solving
- Collaboration
- Initiative
- Growth

**Rating Scale:**
- 5: Exceptional (top 5%)
- 4: Exceeds expectations
- 3: Meets expectations
- 2: Needs improvement
- 1: Unsatisfactory

### Compensation Review

**Annual increases:**
- Market adjustment: 3-5%
- Performance-based: 0-10%
- Promotion: 10-20%

**Equity (for key hires):**
- Early employees (first 10): 0.5-2%
- Key leadership: 1-5%
- Vesting: 4 years, 1-year cliff

## Culture & Retention

### Culture Principles

1. **Client success first** - We win when clients win
2. **Radical candor** - Direct, kind feedback
3. **Ownership** - Act like an owner
4. **Continuous learning** - Always improving
5. **Work-life harmony** - Sustainable pace

### Retention Strategies

**Compensation:**
- Market-rate or above
- Annual reviews
- Performance bonuses (10-20% of base)

**Growth:**
- Learning budget ($2-5K/year)
- Conference attendance
- Internal promotion paths
- Mentorship programs

**Benefits:**
- Health insurance
- 401(k) matching
- Unlimited PTO (with minimums)
- Home office stipend
- Latest equipment

**Recognition:**
- Public shout-outs
- Spot bonuses
- Annual awards
- Client feedback sharing

### Exit Interviews

**When someone leaves:**
- Understand why (compensation, growth, management, other)
- Identify patterns
- Improve based on feedback
- Maintain relationships (boomerang potential)
