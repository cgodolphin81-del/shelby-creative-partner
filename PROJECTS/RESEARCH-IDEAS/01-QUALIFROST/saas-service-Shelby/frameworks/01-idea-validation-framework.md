# SaaS Idea Validation Framework

A systematic approach to validate your SaaS idea before writing a single line of code.

---

## Phase 1: Market Research

### 1.1 Problem Validation

**Questions to Answer:**
- [ ] What specific problem does this solve?
- [ ] Who experiences this problem most acutely?
- [ ] How do they currently solve it?
- [ ] How much does the current solution cost (in time/money)?
- [ ] How frequently do they encounter this problem?

**Validation Methods:**
1. **Customer Interviews** (10-20 target users)
   - Ask about their current workflow
   - Don't pitch your solution initially
   - Listen for pain points and workarounds

2. **Survey Distribution**
   - Target: 100+ responses
   - Platforms: Reddit, Twitter, LinkedIn, niche communities
   - Key metric: % who say they'd pay for a solution

3. **Search Volume Analysis**
   - Google Trends: Is interest growing?
   - Keyword research: Monthly search volume for problem-related terms
   - Forum activity: Reddit, Quora, niche forums

### 1.2 Market Size Assessment

**TAM/SAM/SOM Calculation:**

| Metric | Definition | Your Estimate |
|--------|------------|---------------|
| TAM (Total Addressable Market) | Everyone who could use this | $_____ |
| SAM (Serviceable Addressable Market) | Those you can realistically reach | $_____ |
| SOM (Serviceable Obtainable Market) | Market share in Year 1-3 | $_____ |

**Market Indicators:**
- [ ] Growing industry (YoY growth >10%)
- [ ] Multiple competitors (validates demand)
- [ ] Recent funding in space (investor confidence)
- [ ] Enterprise adoption trends

### 2.2 Competitive Advantage Framework

**Your Differentiators:**
1. **Feature differentiation:** What can you do better?
2. **Price differentiation:** Can you offer better value?
3. **Niche differentiation:** Can you serve an underserved segment?
4. **Experience differentiation:** Better UX, onboarding, support?

**Moat Analysis:**
- [ ] Network effects possible?
- [ ] Data advantages?
- [ ] Switching costs?
- [ ] Brand potential?
- [ ] Technical barriers?

## Phase 3: MVP Scoping

### 3.1 Feature Prioritization Matrix

**Must Have (MVP):**
- Core functionality that solves the primary problem
- Basic user authentication
- Essential integrations
- Payment processing

**Should Have (V2):**
- Quality of life improvements
- Additional integrations
- Advanced features
- Mobile app

**Nice to Have (V3+):**
- Premium features
- AI/automation
- Advanced analytics
- White-label options

### 3.2 MVP Success Metrics

**Pre-Launch:**
- [ ] Landing page conversion rate >5%
- [ ] Waitlist signups: 500+
- [ ] Customer interviews: 20+ positive responses
- [ ] LOIs from potential customers: 10+

**Post-Launch (First 90 Days):**
- [ ] Activation rate: >40%
- [ ] Week 1 retention: >50%
- [ ] Month 1 retention: >30%
- [ ] NPS score: >30
- [ ] Paying conversion: >3%

### 3.3 Build vs. Buy Decision Tree

```
Does this feature differentiate us?
├── YES → Build it
└── NO → Can we use an existing solution?
    ├── YES → Buy/Integrate (Auth0, Stripe, etc.)
    └── NO → Is it core to the experience?
        ├── YES → Build it well
        └── NO → Skip for now
```

## Phase 4: Validation Checklist

### 4.1 Pre-Build Validation

- [ ] 20+ customer interviews completed
- [ ] Problem validated (80%+ say it's a real problem)
- [ ] Willingness to pay confirmed (50%+ would pay)
- [ ] Competitive analysis complete
- [ ] Clear differentiation identified
- [ ] MVP scope defined
- [ ] Technical feasibility confirmed
- [ ] Initial financial projections done

### 4.2 Smoke Test Validation

- [ ] Landing page created
- [ ] Value proposition clear
- [ ] Pricing displayed
- [ ] 500+ visitors
- [ ] 5%+ conversion to waitlist/demo
- [ ] 10+ people request early access

### 4.3 Beta Validation

- [ ] 20-50 beta users onboarded
- [ ] Core workflow tested
- [ ] Feedback incorporated
- [ ] 40%+ weekly active usage
- [ ] 3+ customers willing to pay early
- [ ] Support load manageable

## Red Flags 🚩

Stop and reassess if:
- Can't find 20 people with the problem
- Everyone says "interesting" but no one commits
- Market is shrinking or saturated
- Competitors are well-funded incumbents
- Customer acquisition cost > lifetime value
- No clear path to differentiation
