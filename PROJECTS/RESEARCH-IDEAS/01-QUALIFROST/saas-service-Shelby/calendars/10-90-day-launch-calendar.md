# 90-Day SaaS Launch Calendar

Complete day-by-day launch plan from idea to revenue.

---

## Overview

| Phase | Duration | Focus |
|-------|----------|-------|
| **Phase 1: Foundation** | Days 1-30 | Validate, plan, build MVP |
| **Phase 2: Build & Test** | Days 31-60 | Develop, test, prepare launch |
| **Phase 3: Launch & Grow** | Days 61-90 | Launch, market, optimize |

---

## Phase 1: Foundation (Days 1-30)

### Week 1: Validation & Research

#### Day 1-2: Problem Validation
- [ ] Define target customer
- [ ] Write problem statement
- [ ] List current solutions
- [ ] Identify gaps/opportunities

**Deliverable:** Problem validation document

#### Day 3-5: Customer Interviews
- [ ] Recruit 10 interview candidates
- [ ] Prepare interview script
- [ ] Conduct 5+ interviews
- [ ] Document insights

**Deliverable:** Interview summary report

**Interview Script Template:**
```
1. Tell me about your role/workflow
2. What's the most frustrating part of [relevant task]?
3. How do you currently solve this?
4. How much time/money does this cost you?
5. Have you tried any tools to help? What worked/didn't?
6. If you had a magic wand, what would you change?
7. Would you pay for a solution? How much?
```

#### Day 6-7: Market Research
- [ ] Analyze competitors (5-10)
- [ ] Calculate market size (TAM/SAM/SOM)
- [ ] Identify market trends
- [ ] Define positioning

**Deliverable:** Competitive analysis matrix

**Competitor Analysis Template:**
```
| Competitor | Price | Features | Strengths | Weaknesses | Reviews |
|------------|-------|----------|-----------|------------|---------|
| | | | | | |
```

### Week 2: Solution Design

#### Day 8-10: MVP Scoping
- [ ] List all possible features
- [ ] Prioritize (Must/Should/Could/Won't)
- [ ] Define MVP feature set
- [ ] Create user stories

**Deliverable:** MVP feature document

**Feature Prioritization:**
```
MUST HAVE (MVP)
- Core functionality
- User authentication
- Payment processing
- Basic admin

SHOULD HAVE (V2)
- Advanced features
- Integrations
- Mobile app

COULD HAVE (V3)
- Nice-to-haves
- Premium features

WON'T HAVE (Yet)
- Out of scope for now
```

#### Day 11-12: Technical Architecture
- [ ] Choose tech stack
- [ ] Design database schema
- [ ] Plan integrations
- [ ] Estimate infrastructure needs

**Deliverable:** Technical architecture doc

#### Day 13-14: Business Model
- [ ] Define pricing strategy
- [ ] Calculate unit economics
- [ ] Create financial projections
- [ ] Set success metrics

**Deliverable:** Business model canvas

### Week 3: Brand & Messaging

#### Day 15-17: Brand Development
- [ ] Choose company name
- [ ] Check domain/trademark availability
- [ ] Create logo (or hire designer)
- [ ] Define brand guidelines

**Deliverable:** Brand guidelines document

**Checklist:**
- [ ] Domain secured (.com preferred)
- [ ] Social handles claimed
- [ ] Logo files (SVG, PNG, favicon)
- [ ] Color palette
- [ ] Typography
- [ ] Voice/tone guidelines

#### Day 18-19: Value Proposition
- [ ] Write value proposition statement
- [ ] Create messaging hierarchy
- [ ] Develop elevator pitch
- [ ] Write positioning statement

**Deliverable:** Messaging document

**Value Proposition Template:**
```
For [target customer]
Who [customer need/problem]
[Product name] is a [category]
That [key benefit/solution]
Unlike [competitors]
We [unique differentiation]
```

#### Day 20-21: Landing Page Copy
- [ ] Write headline
- [ ] Write subheadline

- [ ] Write CTA copy

**Deliverable:** Landing page copy doc

### Week 4: Build Foundation

#### Day 22-24: Landing Page
- [ ] Set up Webflow/Framer/Carrd
- [ ] Implement design
- [ ] Add copy
- [ ] Set up analytics
- [ ] Add email capture

**Deliverable:** Live landing page

**Landing Page Must-Haves:**
- [ ] Clear headline
- [ ] Value proposition
- [ ] Benefits (3-5)
- [ ] Social proof (logos, testimonials)
- [ ] CTA (waitlist, early access)
- [ ] Email capture
- [ ] Footer with links

#### Day 25-26: Waitlist Setup
- [ ] Set up email platform (ConvertKit, Mailchimp)
- [ ] Create welcome email sequence
- [ ] Set up automation
- [ ] Test signup flow

**Deliverable:** Email automation live

**Welcome Email Sequence:**
```
Email 1 (Immediate): Welcome + what to expect
Email 2 (Day 2): Problem agitation + your solution
Email 3 (Day 4): Social proof + case study
Email 4 (Day 7): Behind the scenes + building update
Email 5 (Day 10): Launch announcement (when ready)
```

#### Day 27-28: Early Traction
- [ ] Launch on Product Hunt (upcoming)
- [ ] Post on Reddit (relevant subreddits)
- [ ] Share on Twitter/LinkedIn
- [ ] Reach out to network
- [ ] Goal: 100+ waitlist signups

**Deliverable:** Initial traction

## Risk Mitigation

### Common Risks & Solutions

| Risk | Probability | Impact | Mitigation |
|------|-------------|--------|------------|
| Scope creep | High | High | Strict MVP definition |
| Technical debt | Medium | Medium | Code review, testing |
| Low traction | Medium | High | Early validation, iterate |
| Churn | Medium | High | Onboarding, support |
| Cash flow | Medium | High | Monitor burn, pre-sell |
