## Exercise 4: Rate Limiting & Optimization

### Objective
Handle API rate limits gracefully.

### Implementation

**Rate Limit Handling:**
```javascript
// Pseudocode for rate limit handling
async function callAI(prompt) {
  let retries = 0;
  const maxRetries = 3;
  
  while (retries < maxRetries) {
    try {
      const response = await fetch(API_URL, options);
      
      if (response.status === 429) {
        // Rate limited
        const retryAfter = response.headers.get('Retry-After');
        await sleep(retryAfter * 1000);
        retries++;
        continue;
      }
      
      return response.json();
    } catch (error) {
      retries++;
      if (retries >= maxRetries) throw error;
      await sleep(1000 * retries); // Exponential backoff
    }
  }
}
```

# Instructor FAQ

## Common Questions

**Q: What if students have different skill levels?**
A: Use breakout groups, provide advanced challenges for faster learners, pair stronger with weaker students.

**Q: How do I handle technical difficulties during labs?**
A: Have TAs ready to help, provide screenshots as backup, record solutions for common issues.

**Q: What if we run out of time?**
A: Prioritize must-know content, provide recording for rest, offer office hours for questions.

**Q: How do I keep remote students engaged?**
A: Use polls every 10 minutes, cold call (gently), use chat actively, breakout rooms.

**Q: What if a student asks a question I can't answer?**
A: Be honest, commit to finding out, follow up within 24 hours.

## Best Practices

1. **Over-prepare:** Better to have too much than too little
2. **Practice demos:** Never demo live for first time
3. **Build in buffer:** Everything takes longer than expected
4. **Get feedback early:** Mid-class check, not just end
5. **Stay current:** AI tools change fast, update monthly
