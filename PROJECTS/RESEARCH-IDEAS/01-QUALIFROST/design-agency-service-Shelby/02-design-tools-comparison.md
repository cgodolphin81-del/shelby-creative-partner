## 1. Figma

**The industry standard for collaborative design.**

### Pricing
| Plan | Price | Features |
|------|-------|----------|
| Starter | Free | 3 files, unlimited viewers |
| Professional | $15/editor/mo | Unlimited files, version history |
| Organization | $45/editor/mo | Advanced security, design systems |
| Enterprise | $75/editor/mo | SSO, advanced controls |

### Strengths ✅
- **Real-time collaboration** - Multiple designers work simultaneously
- **Browser-based** - No installation, works on any OS
- **Design systems** - Best-in-class component libraries
- **Developer handoff** - Inspect mode, code export
- **Plugin ecosystem** - 10,000+ plugins
- **Auto Layout** - Responsive design made easy
- **Variables** - Design tokens native support
- **FigJam** - Built-in whiteboarding

### Limitations ❌
- Requires internet for full functionality
- Performance can lag with very large files
- Offline mode is limited
- Advanced prototyping less powerful than Protopie

### Best For
- Remote teams requiring collaboration
- Design system development
- Startups to enterprise (scales well)
- Teams wanting single source of truth

### Agency Recommendation: **PRIMARY TOOL**
Use as your core design platform for 90% of projects.

## 2. Sketch

**The Mac-native design pioneer.**

### Pricing
| Plan | Price | Features |
|------|-------|----------|
| Standard | $99/year | Unlimited designs, 1 year updates |
| Business | $20/user/mo | Collaboration, version control |

### Strengths ✅
- **Native Mac app** - Fast, smooth performance
- **Offline-first** - Works without internet
- **Mature ecosystem** - 10+ years of plugins
- **Symbols** - Robust component system
- **Artboards** - Flexible canvas organization
- **One-time purchase** - Standard plan is annual, not subscription

### Limitations ❌
- **Mac only** - No Windows/Linux support
- **Collaboration** - Requires third-party tools (Abstract, Plant)
- **Declining market share** - Industry moving to Figma
- **Handoff** - Requires Zeplin or similar
- **No browser version** - Must install software

### Best For
- Mac-only teams
- Designers preferring native apps
- Offline-heavy workflows
- Legacy project maintenance

### Agency Recommendation: **SECONDARY/LEGACY**
Only use if client requires or team has strong preference. Not recommended for new projects.

## 3. Adobe XD

**Adobe's integrated design solution.**

### Pricing
| Plan | Price | Features |
|------|-------|----------|
| XD Standalone | $9.99/mo | XD only |
| Creative Cloud | $52.99/mo | All Adobe apps |
| Teams | $30/user/mo | Collaboration features |

### Strengths ✅
- **Adobe integration** - Seamless with PS, AI, AE
- **Fast performance** - Native app, optimized
- **Repeat Grid** - Quick list/card creation
- **Voice prototyping** - Unique voice interaction features
- **Responsive Resize** - Auto-adjusting layouts
- **Coediting** - Real-time collaboration (improved)

### Limitations ❌
- **Smaller community** - Less plugins, tutorials
- **Feature stagnation** - Slower innovation vs Figma
- **Market decline** - Adobe focused on Figma acquisition (blocked)
- **Handoff** - Less polished than Figma
- **Cross-platform issues** - Some features Mac-only

### Best For
- Existing Adobe Creative Cloud users
- Teams heavily invested in Adobe ecosystem
- Voice interface projects
- Motion design integration

### Agency Recommendation: **SITUATIONAL**
Use only if client has Adobe enterprise license or specific Adobe workflow requirements.

## 4. Protopie

**Advanced prototyping for complex interactions.**

### Pricing
| Plan | Price | Features |
|------|-------|----------|
| Basic | Free | 1 prototype, limited features |
| Pro | $79/mo | Unlimited prototypes, cloud |
| Enterprise | $199/mo | Team features, SSO |

### Strengths ✅
- **Advanced interactions** - Variables, conditions, triggers
- **Sensor support** - Gyroscope, compass, sound
- **Multi-screen logic** - Complex user flows
- **Native components** - Keyboard, status bar, notch
- **Device testing** - Test on real devices via app
- **Animation** - Timeline-based, physics-based
- **Integration** - Connects with Figma, Sketch, XD

### Limitations ❌
- **Learning curve** - More complex than built-in prototyping
- **Cost** - Expensive for small teams
- **Overkill** - Too advanced for simple prototypes
- **Not for UI design** - Prototyping only, need Figma/Sketch

### Best For
- Complex micro-interactions
- Mobile app prototypes with gestures
- IoT and voice interfaces
- High-fidelity user testing
- Motion-heavy experiences

### Agency Recommendation: **SPECIALIST TOOL**
Use for projects requiring advanced prototyping beyond Figma's capabilities.

## 📊 Agency Tool Stack Recommendation

### Standard Stack (Recommended for Most Agencies)
```
Primary Design: Figma Professional ($15/editor/mo)
Prototyping: Figma (built-in) + Protopie ($79/mo) for complex
Documentation: Zeroheight ($79/mo) or Figma-native
Handoff: Figma Inspect (included)
Whiteboarding: FigJam (included) or Miro ($10/mo)
```

**Monthly Cost for 5-person team: ~$200-300**

### Enterprise Stack (Large Teams, Complex Needs)
```
Primary Design: Figma Organization ($45/editor/mo)
Advanced Prototyping: Protopie Enterprise ($199/mo)
Design Systems: Zeroheight Enterprise ($150/mo)
User Research: UserTesting (custom pricing)
Workshops: Miro Business ($18/mo)
```

**Monthly Cost for 20-person team: ~$1,500-2,500**

### Budget Stack (Solo/Freelance)
```
Primary Design: Figma Starter (Free)
Prototyping: Figma (built-in)
Documentation: Notion (Free)
Handoff: Figma Inspect (free for viewers)
```

**Monthly Cost: $0-20**

## 📈 Migration Considerations

### Moving from Sketch to Figma
1. Audit existing Sketch files
2. Use Sketch-to-Figma plugins
3. Rebuild components as Figma Components
4. Train team on collaboration features
5. Migrate gradually (new projects first)

**Timeline:** 2-4 weeks for small team, 2-3 months for enterprise

### Moving from XD to Figma
1. Export XD assets
2. Rebuild in Figma (no direct import)
3. Leverage Figma's Auto Layout
4. Set up design system structure
5. Developer handoff training

**Timeline:** 3-6 weeks depending on complexity

## 💡 Best Practices

1. **Standardize on one primary tool** - Reduces friction, simplifies hiring
2. **Invest in design system early** - Pays dividends in efficiency
3. **Train developers on handoff tools** - Reduces back-and-forth
4. **Use version control** - Name versions, document changes
5. **Maintain component libraries** - Keep them DRY and documented
6. **Regular tool audits** - Quarterly review of what's working
