# Design Process

End-to-end design methodology from discovery through handoff. A repeatable, scalable process for consistent delivery.

---

## 🔄 Process Overview

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  DISCOVERY  │ →  │   RESEARCH  │ →  │  WIREFRAMES │
│  Week 1-2   │    │  Week 2-4   │    │  Week 4-6   │
└─────────────┘    └─────────────┘    └─────────────┘
                                              ↓
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│   HANDOFF   │ ←  │ PROTOTYPING │ ←  │   MOCKUPS   │
│  Week 10-12 │    │  Week 8-10  │    │  Week 6-8   │
└─────────────┘    └─────────────┘    └─────────────┘
```

**Total Timeline: 10-12 weeks (varies by project scope)**

---

## Phase 1: Discovery

**Understanding the problem before solving it.**

### Duration: 1-2 weeks

### Objectives
- Align on business goals and success metrics
- Understand target users and their needs
- Define project scope and constraints
- Establish working relationships

### Activities

#### Kickoff Workshop (2-3 hours)
**Attendees:** Client stakeholders, design team, PM

**Agenda:**
1. Project background & business context (30 min)
2. Goals & success metrics (30 min)
3. Target audience discussion (30 min)
4. Technical constraints & requirements (30 min)
5. Timeline & deliverables review (30 min)
6. Q&A and next steps (30 min)

**Outputs:**
- Meeting notes
- Initial stakeholder map
- Risk register
- Communication plan

#### Stakeholder Interviews (3-5 interviews, 45 min each)
**Questions to explore:**
- What problem are we solving?
- Who are we solving it for?
- What does success look like?
- What are the biggest risks?
- What constraints exist (technical, business, timeline)?

#### Competitive Review
- Identify 3-5 direct competitors
- Analyze 2-3 adjacent/benchmark products
- Document patterns, opportunities, differentiators

### Deliverables
- [ ] Project brief document
- [ ] Stakeholder interview summaries
- [ ] Competitive analysis deck
- [ ] Initial project timeline
- [ ] Communication protocols

### Approval Gate
**Client sign-off required before proceeding to Research phase**

---

## Phase 2: Research

**Gathering insights to inform design decisions.**

### Duration: 2-3 weeks

### Objectives
- Understand user behaviors, needs, and pain points
- Validate assumptions from discovery
- Identify opportunities for differentiation
- Build empathy across the team

### Activities

#### User Interviews (5-8 participants, 45-60 min each)
**Recruitment criteria:**
- Match target user personas
- Mix of new and experienced users (if applicable)
- Include edge cases where relevant

**Interview guide structure:**
1. Background & context (10 min)
2. Current behaviors & workflows (15 min)
3. Pain points & frustrations (15 min)
4. Reaction to concepts (if applicable) (15 min)
5. Wrap-up & questions (5 min)

**Outputs:**
- Interview recordings (with consent)
- Note summaries per participant
- Affinity diagram of insights
- Key findings report

#### Analytics Review (if existing product)
- Traffic patterns and user flows
- Drop-off points and friction
- Feature usage data
- Support ticket analysis

#### Survey (optional, for quantitative validation)
- Target: 100+ responses
- Focus: Validate qualitative findings
- Tools: Typeform, Google Forms, SurveyMonkey

### Deliverables
- [ ] User personas (3-5 detailed profiles)
- [ ] Current state journey map
- [ ] Research findings report
- [ ] Opportunity areas document
- [ ] Recommendations for design direction

### Approval Gate
**Client review of research findings; alignment on design direction**

---

## Phase 3: Wireframes

**Structuring the experience before visual design.**

### Duration: 2-3 weeks

### Objectives
- Define information architecture
- Establish user flows
- Create low-fidelity layouts
- Validate structure before visual investment

### Activities

#### Information Architecture
- Content audit (if existing product)
- Sitemap creation
- Navigation structure
- Labeling and taxonomy

#### User Flow Diagrams
- Map critical user journeys
- Identify decision points
- Document edge cases
- Define success paths

#### Low-Fidelity Wireframes
**Tools:** Figma, pen/paper, whiteboard

**Focus areas:**
- Layout and hierarchy
- Content prioritization
- Interaction patterns
- Responsive breakpoints

**Fidelity progression:**
1. **Paper sketches** - Rapid ideation
2. **Digital wireframes** - Sharable, editable
3. **Annotated wireframes** - Interaction notes

#### Wireframe Review Sessions
**Internal review:**
- Design team critique
- Technical feasibility check
- Alignment with research insights

**Client review:**
- Walkthrough of key flows
- Gather feedback on structure
- Iterate based on input

### Deliverables
- [ ] Sitemap
- [ ] User flow diagrams
- [ ] Low-fidelity wireframes (all screens)
- [ ] Wireframe annotations
- [ ] Responsive breakpoint documentation

### Approval Gate
**Client sign-off on wireframes before visual design begins**

---

## Phase 4: Mockups

**Bringing the design to life visually.**

### Duration: 2-3 weeks

### Objectives
- Apply visual design language
- Create high-fidelity screens
- Establish visual hierarchy
- Build component library

### Activities

#### Visual Direction
**Mood boards (2-3 directions):**
- Color palettes
- Typography options
- Imagery style
- UI element examples

**Client selection:**
- Present mood boards
- Discuss rationale
- Select direction or hybrid

#### High-Fidelity Design
**Screen design progression:**
1. Key screens first (homepage, primary flows)
2. Secondary screens
3. Edge cases and states
4. All responsive breakpoints

**Design considerations:**
- Accessibility (color contrast, text size)
- Brand consistency
- Platform conventions (iOS, Android, Web)
- Performance implications

#### Component Library
**Build in Figma:**
- Buttons and CTAs
- Form elements
- Navigation components
- Cards and containers
- Modals and overlays
- Typography styles
- Color styles

### Deliverables
- [ ] High-fidelity mockups (all screens)
- [ ] Component library
- [ ] Style guide (colors, typography, spacing)
- [ ] Responsive variants
- [ ] State documentation (hover, active, disabled, error)

### Approval Gate
**Client approval of visual design before prototyping**

---

## Phase 5: Prototyping

**Making the design interactive for validation.**

### Duration: 1-2 weeks

### Objectives
- Create interactive experience
- Validate user flows
- Identify interaction issues
- Prepare for usability testing

### Activities

#### Prototype Fidelity Decision
| Fidelity | Use Case | Tool |
|----------|----------|------|
| Low | Internal testing, early validation | Figma basic links |
| Medium | Client review, stakeholder demos | Figma smart animate |
| High | Usability testing, developer reference | Figma + Protopie |

#### Interaction Design
**Define:**
- Transitions and animations
- Micro-interactions
- Gesture behaviors (mobile)
- Loading states
- Error states
- Empty states

#### Prototype Build
**Figma prototyping:**
- Connect screens with flows
- Add transitions
- Configure overlays and modals
- Set up variants for states

**Protopie (for advanced):**
- Variables and conditions
- Sensor interactions
- Complex animations
- Multi-screen logic

#### Internal Testing
- Design team walkthrough
- Identify broken flows
- Fix interaction issues
- Optimize for performance

### Deliverables
- [ ] Interactive prototype (shareable link)
- [ ] Prototype documentation
- [ ] Interaction specifications
- [ ] Animation guidelines (if applicable)

### Approval Gate
**Client demo and approval; ready for usability testing or handoff**

---

## Phase 6: Handoff

**Ensuring smooth transition to development.**

### Duration: 1-2 weeks (plus ongoing support)

### Objectives
- Provide developers with clear specifications
- Enable accurate implementation
- Establish support channel for questions
- Ensure design integrity in build

### Activities

#### Handoff Preparation
**Figma file organization:**
- Clean, labeled layers
- Organized pages and frames
- Component documentation
- Style definitions

**Developer documentation:**
- Design system documentation
- Component usage guidelines
- Interaction specifications
- Asset export list

#### Handoff Meeting
**Attendees:** Design team, development team, PM

**Agenda:**
1. Project overview (15 min)
2. Design system walkthrough (30 min)
3. Key flows demo (30 min)
4. Technical Q&A (30 min)
5. Support process review (15 min)

#### Developer Access
- Figma viewer access for all developers
- Zeplin/Zeroheight access (if used)
- Asset library access
- Contact information for design questions

#### Support During Build
**Recommended cadence:**
- Weekly check-ins (30 min)
- Slack/Teams channel for questions
- Design review of staged builds
- Bug triage participation

### Deliverables
- [ ] Organized Figma file
- [ ] Design system documentation
- [ ] Asset package (icons, images, fonts)
- [ ] Interaction specifications
- [ ] Developer contact document
- [ ] Support schedule

### Approval Gate
**Development team confirms understanding; design support begins**

## 🎯 Process Adaptation by Project Type

### Small Project (4-6 weeks)
- Combine Discovery + Research (1 week)
- Streamlined wireframes (1 week)
- Mockups + Prototype combined (2-3 weeks)
- Light handoff (1 week)

### Enterprise Project (12-20 weeks)
- Extended Discovery (2-3 weeks)
- Comprehensive Research (4 weeks)
- Iterative wireframes (3-4 weeks)
- Phased mockups (4-6 weeks)
- Advanced prototyping (2-3 weeks)
- Formal handoff + support (2-4 weeks)

### Design System Project (8-12 weeks)
- Audit existing designs (2 weeks)
- Token definition (2 weeks)
- Component design (4-6 weeks)
- Documentation (2 weeks)
- Training + handoff (2 weeks)
