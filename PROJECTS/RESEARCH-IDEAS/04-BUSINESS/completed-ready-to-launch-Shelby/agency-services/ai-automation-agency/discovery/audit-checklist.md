# Client Audit Checklist

## Pre-Audit Preparation

### Information Gathering
- [ ] Company website reviewed
- [ ] LinkedIn company page reviewed
- [ ] Key stakeholder LinkedIn profiles reviewed
- [ ] Industry research completed
- [ ] Competitor automation analysis (if applicable)
- [ ] Pre-discovery questionnaire received and reviewed

### Technical Preparation
- [ ] Discovery call scheduled with key stakeholders
- [ ] Miro/Mural board prepared for workflow mapping
- [ ] Recording consent obtained (if recording sessions)
- [ ] Note-taking system ready
- [ ] ROI calculator template prepared

---

## Audit Areas

### 1. Business Context
- [ ] Company size and structure documented
- [ ] Key departments and functions identified
- [ ] Decision-making process understood
- [ ] Strategic priorities for the year identified
- [ ] Current initiatives and projects understood
- [ ] Pain points ranked by severity

### 2. Technology Stack
- [ ] CRM system identified and version noted
- [ ] ERP/accounting software identified
- [ ] Project management tools documented
- [ ] Communication tools (email, chat, video) documented
- [ ] Marketing tools documented
- [ ] Customer support tools documented
- [ ] HR/People systems documented
- [ ] Custom or legacy systems identified
- [ ] API availability assessed for key systems
- [ ] Data export capabilities assessed

### 3. Workflow Analysis

#### For Each Priority Workflow:
- [ ] Trigger identified
- [ ] All steps documented
- [ ] Roles and responsibilities mapped
- [ ] Tools used at each step noted
- [ ] Time per step estimated
- [ ] Volume/frequency documented
- [ ] Pain points identified
- [ ] Exceptions and edge cases noted
- [ ] Dependencies identified
- [ ] Outputs and success criteria defined

#### Common Workflows to Audit:
- [ ] Lead capture to CRM entry
- [ ] Lead qualification and nurturing
- [ ] Sales proposal generation
- [ ] Contract creation and approval
- [ ] Customer onboarding
- [ ] Order processing
- [ ] Invoice generation and payment
- [ ] Customer support ticket handling
- [ ] Employee onboarding
- [ ] Content creation and approval
- [ ] Reporting and analytics
- [ ] Inventory management
- [ ] Vendor management

### 4. Data Assessment
- [ ] Data sources identified
- [ ] Data formats documented (CSV, JSON, API, etc.)
- [ ] Data quality issues noted
- [ ] Data volume assessed
- [ ] Data update frequency understood
- [ ] Data ownership and access clarified
- [ ] Data security requirements identified
- [ ] Compliance requirements noted (GDPR, HIPAA, etc.)

### 5. Integration Requirements
- [ ] Must-have integrations identified
- [ ] Nice-to-have integrations noted
- [ ] API documentation requested for key systems
- [ ] Authentication methods understood (OAuth, API keys, etc.)
- [ ] Rate limits and quotas assessed
- [ ] Webhook capabilities assessed
- [ ] Custom integration needs identified

### 6. Team & Change Management
- [ ] Key stakeholders identified
- [ ] End users identified
- [ ] Technical capabilities of team assessed
- [ ] Change champions identified
- [ ] Potential resistors identified
- [ ] Training needs assessed
- [ ] Communication preferences understood

### 7. Success Metrics
- [ ] Primary KPIs identified
- [ ] Secondary metrics noted
- [ ] Current baseline metrics captured
- [ ] Target metrics defined
- [ ] Reporting requirements understood
- [ ] Dashboard needs assessed

### 8. Constraints & Requirements
- [ ] Budget range confirmed
- [ ] Timeline requirements documented
- [ ] Technical constraints identified
- [ ] Security requirements documented
- [ ] Compliance requirements documented
- [ ] Scalability requirements understood
- [ ] Maintenance capabilities assessed

## Red Flags to Watch For

### Technical Red Flags
- ⚠️ Legacy systems with no API access
- ⚠️ Highly customized systems with no documentation
- ⚠️ Data silos with no clear ownership
- ⚠️ Poor data quality requiring extensive cleanup
- ⚠️ Security requirements beyond our capability
- ⚠️ Compliance requirements we can't meet

### Organizational Red Flags
- ⚠️ No clear decision-maker
- ⚠️ History of failed automation projects
- ⚠️ Strong resistance to change
- ⚠️ Unrealistic expectations about timeline or cost
- ⚠️ No internal champion
- ⚠️ Budget not aligned with scope

### Process Red Flags
- ⚠️ Processes not documented or understood
- ⚠️ Constantly changing requirements
- ⚠️ Processes that require human judgment at every step
- ⚠️ Exception-heavy workflows (>30% exceptions)
- ⚠️ Dependencies on external parties outside their control

## Post-Audit Actions

- [ ] Audit report completed and reviewed
- [ ] Internal team debrief conducted
- [ ] Go/No-Go decision made
- [ ] If Go: Proposal development initiated
- [ ] If No-Go: Polite decline with referrals if appropriate
- [ ] Audit findings added to CRM
- [ ] Follow-up scheduled if needed

---

## Audit Quality Checklist

Before finalizing audit:
- [ ] All stakeholders interviewed
- [ ] All priority workflows mapped
- [ ] Technology stack fully documented
- [ ] Pain points quantified where possible
- [ ] ROI opportunities identified
- [ ] Constraints and risks documented
- [ ] Recommendations are actionable
- [ ] Report is clear and client-ready
